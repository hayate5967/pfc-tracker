# 機能仕様書 — PFCカロリー記録アプリ

## 1. 概要

### 目的
食事の写真を1枚撮るだけで、事前に登録しておいた食べ物から自動分類し、PFC（たんぱく質・脂質・炭水化物）とカロリーを記録するWebアプリ。

### カロミル等との差別化
- **クローズドセット分類**：世界中の食べ物ではなく「ユーザーが登録した食べ物のみ」から分類するため精度が高い
- **ブラウザ完結**：画像認識はブラウザ内のTF.jsで処理。撮影時にAPIコスト不要

---

## 2. ユーザー

- 単一ユーザー（認証なし）
- 主な利用端末：スマートフォン（モバイルファースト設計）

---

## 3. 機能一覧

### 3.1 食べ物登録

| 項目 | 内容 |
|------|------|
| 名前 | 任意の文字列（例：ご飯、鶏胸肉） |
| たんぱく質 | 100gあたりのg数（小数可） |
| 脂質 | 100gあたりのg数（小数可） |
| 炭水化物 | 100gあたりのg数（小数可） |
| カロリー | 100gあたりのkcal（小数可） |
| 重さモード | `固定`（常に一定g）または`ボタン選択`（複数の選択肢から選ぶ） |
| 固定重量 | 重さモードが「固定」のとき入力（例：150g） |
| 重さ選択肢 | 重さモードが「ボタン選択」のとき、複数設定可（例：50, 100, 150） |
| 登録画像 | 1枚以上（多いほど分類精度が向上）。カメラで撮影 |

#### 登録フロー
```
名前・PFC入力 → 重さモード設定 → 写真撮影（1枚以上）→ 保存
```

#### 精度向上のヒント
- 同じ食べ物を異なる角度・照明・盛り付けで複数枚登録する
- 最低1枚あれば動作するが、3〜5枚以上を推奨

### 3.2 食べ物編集・削除

- 名前・PFC・カロリーを編集可能（重さモードは現在変更不可）
- 登録画像の追加・個別削除が可能
- 食べ物の削除（関連するfood_imagesも連鎖削除される）

### 3.3 食事記録

#### 記録フロー
```
カメラを起動 → 食事を撮影 → 自動検出（COCO-SSD + MobileNet KNN）
→ 検出結果確認（全品目の重さを選択）→ まとめて保存
```

#### 自動検出の仕様
1. **COCO-SSD**で写真内の物体を検出（信頼度0.3以上）
2. 各検出領域を5%のパディングつきでクロップ
3. 各クロップに**MobileNet + KNN**を適用して登録済み食品と照合
4. 類似度0.5以上かつ重複なしの結果を採用
5. COCO-SSDが何も検出しない場合は画像全体をKNN分類（フォールバック）

#### 重さ選択
- `固定`モードの食品：自動的に固定重量が適用（ボタンを押すだけ）
- `ボタン選択`モードの食品：選択肢ボタンが表示される

#### 保存内容
各食品について以下を`meal_logs`に保存：
- 食品ID・重さ・たんぱく質g・脂質g・炭水化物g・カロリー・記録日時

### 3.4 ダッシュボード（ホーム）

- 今日の日付表示
- 今日の合計 PFC・カロリーをプログレスバーで表示（目標対比）
- 今日の食事ログ一覧（時刻・食品名・重量・PFC・カロリー）
- 各ログの個別削除

### 3.5 目標設定

設定可能な値：
- 1日のカロリー目標（kcal）
- 1日のたんぱく質目標（g）
- 1日の脂質目標（g）
- 1日の炭水化物目標（g）

空白のままにすると該当項目は「目標なし」（プログレスバー非表示）

### 3.6 食事履歴

- 過去の食事ログを日付グループで表示（新しい日付順）
- 最大200件取得
- 1日の合計カロリーを日付ヘッダーに表示
- 各ログの個別削除

## 5. 非機能要件

### パフォーマンス
- MobileNet + COCO-SSDの初回ロード：10〜20秒（以降はメモリキャッシュ）
- カメラ起動からの検出・分類：5〜15秒

### 対応環境
- モダンブラウザ（Chrome / Safari / Firefox 最新版）
- HTTPSが必要（カメラAPIの制約。localhostは例外）
- スマートフォン推奨（PCでも動作可）

### データ永続化
- Supabase（PostgreSQL）にすべてのデータを保存
- 食品画像は Supabase Storage に保存
- ブラウザのキャッシュやLocalStorageは使用しない

---

## 6. 制約・注意事項

- **認証なし**：データは誰でもアクセス可能。個人利用を想定
- **RLS無効**：Supabaseの行レベルセキュリティは無効化している
- **COCO-SSDの限界**：検出できる物体は80カテゴリ（食品関連は約10〜15）。別々の器に盛り付けた食事が最もよく機能する
- **類似度閾値**：デフォルト0.5。登録画像が少ない場合は誤分類の可能性がある
